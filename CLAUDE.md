# Claude Code Instructions

## Project Context

AI Agent chatbot comparing two architectures on the SWE-QA dataset:

- **ReAct Agent**: Reason-then-Act loop
- **Deep Agent**: Hierarchical architecture with task decomposition and sub-agents

Goal: Academic paper comparing performance and efficiency of both architectures.

## Core Rule: No Manual Coding

All code must be generated by prompting Claude Code. Never write code manually.

## Package Manager Rule

Always use `uv` for Python dependencies, never `pip`.

```
uv add <package>
uv run python main.py
```

## MCP Servers

The following MCP servers are configured and must be used for documentation lookups:

| Server | Transport |
|---|---|
| DeepWiki MCP | `https://mcp.deepwiki.com/mcp` |
| LangChain MCP | `npx -y langchain-mcp` |
| GitHub MCP | `npx -y @modelcontextprotocol/server-github` |

## MCP Usage Rules

- **Always** query DeepWiki MCP and LangChain MCP for up-to-date documentation before writing LangChain, LangGraph, or Deep Agents code.
- Never rely on stale local docs or training-data knowledge for these libraries — they change frequently.
- Before implementing any agent pattern (ReAct, Deep Agent, tool calling, memory, checkpointing), fetch the latest API from LangChain MCP or DeepWiki MCP first.

## Agent Architecture Guidelines

- **ReAct Agent**: Use `create_react_agent` — verify latest syntax via LangChain MCP.
- **Deep Agent**: Use `create_deep_agent` from `deepagents` — verify sub-agent patterns, middleware stack, and tool signatures via DeepWiki MCP.
- For both: handle tool calling, prompt templating, and memory according to up-to-date docs.

## File Editing Rules

- Remove HTML comments when editing any file.
- Keep `.env` files out of git — secrets go in `.env`, example values in `.env.example`.
